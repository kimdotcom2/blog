##  SQL에서 DDL과 DML의 차이점을 설명하고, 각각의 대표적인 명령어들의 용도를 설명하세요.

DDL(Data Definition Language)과 DML(Data Manipulation Language)은 SQL에서 데이터베이스를 관리하고 조작하는 데 사용되는 두 가지 주요 언어이다.

DDL은 데이터베이스의 구조를 정의하고 변경하는 데 사용된다. 
DML은 데이터베이스 내의 데이터를 조회 또는 조작하는 데 사용된다.

DDL의 대표적인 명령어들은 다음과 같다.

1. CREATE: 새로운 데이터베이스 객체(테이블, 인덱스, 뷰 등)를 생성하는 데 사용된다.
2. ALTER: 기존 데이터베이스 객체의 구조를 변경하는 데 사용된다.
3. DROP: 기존 데이터베이스 객체를 삭제하는 데 사용된다.
4. TRUNCATE: 테이블의 모든 데이터를 삭제하지만, 테이블 구조는 유지하는 데 사용된다.

DML의 대표적인 명령어들은 다음과 같다.

1. SELECT: 데이터베이스에서 데이터를 조회하는 데 사용된다.
2. INSERT: 새로운 데이터를 테이블에 삽입하는 데 사용된다.
3. UPDATE: 기존 데이터를 수정하는 데 사용된다.
4. DELETE: 기존 데이터를 삭제하는 데 사용된다.

## 역정규화가 필요한 상황과 적용 시 고려해야 할 사항, 그리고 역정규화를 적용할 때의 장단점을 설명해주세요.

역정규화는 데이터베이스 설계에서 정규화된 구조를 일부러 비정규화하여 성능을 향상시키기 위해 사용된다. 역정규화가 필요한 상황은 다음과 같다.

1. 성능 향상: 복잡한 조인 연산이 빈번하게 발생하는 경우, 역정규화를 통해 데이터 중복을 허용하여 조회 성능을 향상시킬 수 있다.
2. 읽기 중심의 애플리케이션: 읽기 작업이 많은 애플리케이션에서는 역정규화를 통해 데이터 접근 속도를 높일 수 있다.
3. 데이터 분석: 대규모 데이터 분석/통계 작업에서 빠른 조회가 필요할 때 역정규화를 고려할 수 있다.

### 역정규화의 단점

1. 데이터 중복: 역정규화는 데이터 중복을 허용하므로, 데이터 정합성을 유지하기 어려울 수 있다.
2. 정합성 유지: 데이터가 여러 곳에 중복 저장되므로, 데이터 변경 시 모든 관련 데이터를 일관되게 업데이트해야 한다.(배치, 트리거 등으로 해결 가능)


### 역정규화 적용 시 고려해야 할 사항

1. 병목 구간을 파악하고 해당 부분애먼 역정규화를 적용한다.
2. 데이터 정합성 유지를 위해 트리거, 배치 작업, 애플리케이션 작업 등을 활용할 수 있는지 검토한다.
3. 요구사항의 변화에 유연하게 대응할 수 있는지 고려한다.
