## Spring Cache에서 @Cacheable, @CachePut, @CacheEvict의 차이점과 각각을 어떤 상황에서 사용하는 것이 적절한지 설명해주세요.

스프링 캐시에서 @Cacheable, @CachePut, @CacheEvict는 캐시 관리를 위한 어노테이션이다.

@Cacheable은 메서드의 결과를 캐시에 저장하고, 동일한 인자가 들어올 때 캐시된 값을 반환한다. 
주로 자주 호출되는 메서드에서 사용하여 성능을 향상시키는 데 적합하다.

인자는 변하지 않지만 결과가 자주 조회되는 경우에 사용한다.

또한 어노테이션의 설정으로 캐시 이름과 키를 지정할 수 있다.

@CachePut은 메서드가 호출될 때마다 캐시에 값을 저장한다. 
즉, 메서드가 실행될 때마다 캐시를 갱신한다. 
따라서, 데이터가 변경될 때마다 캐시를 업데이트해야 하는 경우에 적합하다.

예를 들어, 데이터베이스에 새로운 데이터를 저장하거나 기존 데이터를 수정하는 메서드에서 사용된다.

@CacheEvict는 캐시에서 특정 항목을 제거하는 데 사용된다. 
데이터가 삭제되거나 변경되어 더 이상 유효하지 않은 캐시 항목을 제거할 때 적합하다. 
예를 들어, 데이터베이스에서 데이터를 삭제하는 메서드에서 사용된다.



## 로컬 캐시와 분산 캐시의 개념 차이와 각각의 장단점, 그리고 실무에서 어떤 기준으로 선택해야 하는지 설명해주세요.

로컬 캐시는 애플리케이션 인스턴스 내에서만 접근 가능한 캐시이다. 
즉, 각 인스턴스마다 독립적인 캐시를 유지한다.

장점:
- 빠른 접근 속도: 메모리 내에 존재하므로 매우 빠르다.
- 간단한 설정: 별도의 인프라 없이 애플리케이션 내에서 쉽게 구현할 수 있다.
- 낮은 지연 시간: 네트워크 호출이 없으므로 지연 시간이 적다.
- 낮은 비용: 별도의 캐시 서버가 필요 없으므로 비용이 절감된다.

단점:
- 데이터 일관성 문제: 여러 인스턴스가 있을 경우 캐시 데이터가 일관되지 않을 수 있다.
- 확장성 제한: 인스턴스가 늘어날수록 캐시 관리가 어려워진다.
- 메모리 제한: 각 인스턴스의 메모리 용량에 제한이 있다.

분산 캐시는 여러 애플리케이션 인스턴스가 공유하는 중앙 집중식 캐시 시스템이다.

장점:
- 데이터 일관성: 모든 인스턴스가 동일한 캐시 데이터를 공유하므로 일관성이 유지된다.
- 확장성: 캐시 서버를 추가하여 쉽게 확장할 수 있다.
- 대용량 데이터 처리: 더 큰 캐시 용량을 제공할 수 있다.
- 고가용성: 분산 시스템을 통해 장애 발생 시에도 캐시 접근이 가능하다.

단점:
- 네트워크 지연: 네트워크 호출이 필요하므로 접근 속도가 느릴 수 있다.
- 복잡한 설정: 별도의 캐시 서버와 네트워크 구성이 필요하다.
- 비용 증가: 별도의 인프라와 관리 비용이 발생할 수 있다.
