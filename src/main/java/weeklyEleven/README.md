## 위클리 페이퍼 11

### AWS RDS를 활용하는 주요 이점과 EC2에 직접 데이터베이스를 설치하여 운영하는 것과 비교했을 때의 차별점에 대해 설명해주세요. 그리고 RDS를 사용하는 것이 적합하지 않을 수 있는 상황도 함께 언급해주세요.

AWS RDS를 사용하는 주요 이점은 다음과 같다.

1. 운영 간소화
DB 인스턴스의 프로비저닝, 패치 적용, 백업, 복구 등 관리 작업이 자동화되어 운영 부담이 크게 줄어든다.

2. 자동 백업 및 복구
RDS는 자동 백업 기능을 제공하여 데이터 손실 위험을 최소화하며, 필요 시 특정 시점으로 복구할 수 있다.

3. 확장성
RDS는 수요에 따라 쉽게 스토리지와 컴퓨팅 리소스를 확장할 수 있어 트래픽 변화에 유연하게 대응할 수 있다.

4. 고가용성 및 내구성
멀티 AZ 배포를 통해 장애 발생 시 자동으로 대체 인스턴스로 전환되어 서비스 중단을 최소화한다.

5. 보안
RDS는 VPC, IAM, KMS 등 AWS의 다양한 보안 기능과 통합되어 데이터베이스 보안을 강화한다.

하지만 다음과 같은 경우에는 RDS 사용이 적합하지 않을 수 있다.

1. 맞춤형 설정 필요
특정 데이터베이스 설정이나 확장이 필요할 경우, RDS의 제한된 커스터마이징 옵션이 문제가 될 수 있다.

2. 비용 문제
장기적으로 대규모 데이터베이스를 운영할 경우, RDS의 비용이 EC2에 직접 설치하는 것보다 높을 수 있다.

3. 특정 데이터베이스 엔진 미지원
RDS가 지원하지 않는 데이터베이스 엔진을 사용해야 하는 경우에는 RDS를 사용할 수 없다.

4. 낮은 레벨의 접근 필요
데이터베이스 서버에 대한 완전한 제어가 필요한 경우, RDS의 관리형 서비스 특성상 제한이 있을 수 있다.


### GitHub Actions 워크플로우에서 사용할 수 있는 다양한 트리거(Trigger) 유형을 설명하고, 각 트리거 유형이 적합한 CI/CD 시나리오에 대해 설명하세요.

GitHub Actions 워크플로우에서 사용할 수 있는 주요 트리거 유형은 다음과 같다.`

1. push
코드가 특정 브랜치에 푸시될 때 워크플로우가 실행된다. 주로 코드 변경 사항을 빌드하고 테스트하는 CI 시나리오에 적합하다.

2. pull_request
풀 리퀘스트가 생성되거나 업데이트될 때 워크플로우가 실행된다. 코드 리뷰와 병합 전 테스트를 자동화하는 데 유용하다.

3. schedule
지정된 시간 간격으로 워크플로우가 실행된다. 정기적인 빌드, 테스트, 배포 작업에 적합하다.

4. workflow_dispatch
사용자가 수동으로 워크플로우를 실행할 수 있도록 한다. 특정 상황에서 수동 배포나 테스트가 필요할 때 유용하다.

5. release
새 릴리스가 생성될 때 워크플로우가 실행된다. 릴리스 빌드 및 배포 자동화에 적합하다.
